# ПОЛНЫЙ СПИСОК СОКРАЩЕНИЙ

## Основные сокращения курса

| Сокращение | Английское значение | Перевод на русский | Пояснение |
|------------|---------------------|-------------------|-----------|
| **АБС** | Automated Banking System | Автоматизированная банковская система | Основная учётная система банка, где ведутся счета, проводки, кредиты (аналог core banking system) |
| **AML** | Anti-Money Laundering | Противодействие легализации доходов | Процесс и система мониторинга подозрительных операций |
| **API** | Application Programming Interface | Интерфейс программирования приложений | Способ взаимодействия между системами |
| **BCBS** | Basel Committee on Banking Supervision | Базельский комитет по банковскому надзору | Международный орган, разрабатывающий стандарты банковского регулирования |
| **BI** | Business Intelligence | Бизнес-аналитика | Системы для сбора, хранения и анализа данных для принятия решений (дашборды, отчёты) |
| **BKI** | Credit History Bureau | Бюро кредитных историй | Внешняя организация, хранящая данные о кредитной истории заёмщиков |
| **CDMP** | Certified Data Management Professional | Сертифицированный специалист по управлению данными | Профессиональная сертификация DAMA International |
| **CFT** | Combating the Financing of Terrorism | Противодействие финансированию терроризма | Часть комплаенс-процедур банка |
| **COPQ** | Cost of Poor Data Quality | Стоимость низкого качества данных | Экономические потери от дефектов данных (штрафы, ручные исправления, упущенная выгода) |
| **CRM** | Customer Relationship Management | Управление взаимоотношениями с клиентами | Система для учёта клиентов, их контактов, анкет, взаимодействий |
| **DAMA** | Data Management Association | Ассоциация управления данными | Международная профессиональная организация, автор стандарта DMBOK |
| **DAMA DMBOK2** | Data Management Body of Knowledge (2nd edition) | Свод знаний по управлению данными (2-е издание) | Основной профессиональный стандарт в области управления данными |
| **DDS** | Detail Data Store | Детальное хранилище данных | Слой DWH, где данные хранятся на уровне отдельных записей/транзакций |
| **DQ** | Data Quality | Качество данных | Степень пригодности данных для использования; одна из областей знаний DAMA |
| **DQAF** | Data Quality Assessment Framework | Структура оценки качества данных | Методология оценки качества данных Международного Валютного Фонда (МВФ) |
| **DWH** | Data Warehouse | Хранилище данных | Централизованное хранилище интегрированных данных из различных источников |
| **EAD** | Exposure at Default | Объём обязательств на момент дефолта | Параметр кредитного риска по Базельским соглашениям — сумма, которую должен клиент в момент дефолта |
| **ETL** | Extract, Transform, Load | Извлечение, преобразование, загрузка | Процесс переноса данных из источников в хранилище |
| **FATCA** | Foreign Account Tax Compliance Act | Закон о налогообложении иностранных счетов (США) | Американский закон, требующий от банков раскрытия информации о счетах граждан США |
| **GFS** | Government Finance Statistics | Статистика государственных финансов | Один из секторов DQAF МВФ |
| **ICT** | Information and Communication Technologies | Информационные и коммуникационные технологии | Общий термин для IT-инфраструктуры |
| **IRB** | Internal Ratings-Based approach | Подход на основе внутренних рейтингов | Метод расчёта кредитного риска по Базелю II/III, требующий высокого качества данных |
| **IS** | Information Systems | Информационные системы | Общий термин для любых систем сбора, обработки и хранения данных |
| **ITIL** | Information Technology Infrastructure Library | Библиотека инфраструктуры информационных технологий | Стандарт управления IT-услугами и инцидентами |
| **ITSM** | IT Service Management | Управление IT-услугами | Дисциплина управления IT-инфраструктурой |
| **KPI** | Key Performance Indicator | Ключевой показатель эффективности | Измеримая метрика для оценки успешности процесса |
| **LGD** | Loss Given Default | Потери при дефолте | Параметр кредитного риска — доля потерь от суммы обязательств при дефолте |
| **LMS** | Learning Management System | Система управления обучением | Платформа для онлайн-курсов и учебных материалов |
| **MDM** | Master Data Management | Управление мастер-данными | Система и процесс поддержания единой, точной версии ключевых данных (клиенты, продукты, справочники) |
| **METU** | Middle East Technical University | Ближневосточный технический университет | Университет в Анкаре, Турция (место защиты диссертации) |
| **MIS** | Management Information Systems | Информационные системы управления | Системы для подготовки управленческой отчётности |
| **ODS** | Operational Data Store | Операционное хранилище данных | Промежуточный слой данных между источниками и DWH; хранит свежие, близкие к операционным данные |
| **PD** | Probability of Default | Вероятность дефолта | Параметр кредитного риска — вероятность того, что клиент не сможет обслуживать долг |
| **PODFT** | Anti-Money Laundering and Counter-Financing of Terrorism | ПОД/ФТ — Противодействие легализации доходов и финансированию терроризма | Российский термин, соответствующий AML/CFT |
| **RAG** | Red-Amber-Green | Красно-жёлто-зелёный (светофор) | Система цветовой индикации качества данных |
| **RAROC** | Risk-Adjusted Return on Capital | Риск-скорректированная доходность капитала | Метрика эффективности использования капитала с учётом рисков |
| **RCA** | Root Cause Analysis | Анализ корневых причин | Методика выявления первопричины проблемы, а не её симптомов |
| **RDM** | Reference Data Management | Управление ссылочными (справочными) данными | Управление неизменными или редко изменяющимися справочниками |
| **ROSC** | Reports on Observance of Standards and Codes | Отчёты о соблюдении стандартов и кодексов | Отчёты МВФ о соответствии стран международным стандартам, включая модуль данных |
| **RWA** | Risk-Weighted Assets | Активы, взвешенные по уровню риска | Показатель, используемый для расчёта достаточности капитала банка |
| **SDDS** | Special Data Dissemination Standard | Специальный стандарт распространения данных | Стандарт МВФ для публикации макроэкономических данных |
| **SLA** | Service Level Agreement | Соглашение об уровне обслуживания | Договорённость между поставщиком и потребителем данных о целевых показателях качества |
| **TDQM** | Total Data Quality Management | Комплексное управление качеством данных | Подход, циклически повторяющий этапы: определение, измерение, анализ, улучшение |
| **TIN** | Taxpayer Identification Number | Идентификационный номер налогоплательщика | Налоговый номер (аналог ИНН в США) |
| **TQM** | Total Quality Management | Всеобщее управление качеством | Управленческая концепция качества, применённая к данным (TDQM) |
| **TTR** | Time to Resolve | Время до разрешения | Метрика SLA — время от создания инцидента до его полного решения |
| **TTQ** | Time to Qualify | Время до принятия в работу | Метрика SLA — время от создания инцидента до начала работ по нему |
| **ЦБ** | Central Bank | Центральный банк | Регулятор банковской системы (в РФ — Банк России / ЦБ РФ) |

## Расшифровка уровней данных (слои DWH / архитектуры данных)

| Сокращение | Английское значение | Перевод на русский | Пояснение |
|------------|---------------------|-------------------|-----------|
| **ODS** | Operational Data Store | Операционное хранилище данных | Слой, куда данные загружаются из источников (АБС, CRM) в почти неизменном виде. Данные «как есть» |
| **DDS** | Detail Data Store | Детальное хранилище данных | Слой очищенных, согласованных, нормализованных данных |
| **DM** | Data Mart | Витрина данных | Специализированный срез данных для конкретной задачи (например, витрина для отчётности ЦБ) |
| **DWH** | Data Warehouse | Хранилище данных (целиком) | Включает ODS, DDS и витрины как единую среду |

## Роли и организационные термины (DAMA)

| Сокращение | Английское значение | Перевод на русский | Пояснение |
|------------|---------------------|-------------------|-----------|
| **CDO** | Chief Data Officer | Директор по управлению данными | Высшее должностное лицо, ответственное за управление данными в организации |
| **Data Owner** | Data Owner | Владелец данных | Бизнес-роль, отвечающая за данные как за актив: определяет требования к качеству, выделяет бюджет, решает спорные вопросы |
| **Data Steward** | Data Steward | Дата-стюард | Операционная роль, отвечающая за качество данных на ежедневной основе: исправляет записи, инициирует инциденты, взаимодействует с бизнес-пользователями |
| **Technical Steward** | Technical Steward | Технический стюард | Техническая роль (инженер данных), отвечающая за пайплайны, ETL, инфраструктуру качества данных |

## Как читать таблицу

1. **При чтении курса** встречается сокращение → находите в таблице по алфавиту.
2. **Английское значение** — полная расшифровка на языке оригинала.
3. **Перевод** — официальный или общепринятый русскоязычный термин.
4. **Пояснение** — краткое описание, что это за термин и где он используется.

**Примеры использования в курсе:**
- *ODS* — «Данные сначала попадают в ODS (операционное хранилище), где хранятся в виде, близком к исходному».
- *RAG* — «Цветовая индикация RAG (светофор) отражает пороговые значения качества: красный — неприемлемо, жёлтый — допустимо, зелёный — хорошо».
- *BCBS 239* — «Документ Базельского комитета (BCBS 239) предъявляет требования к агрегированию данных о рисках».

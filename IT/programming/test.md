## test
- git checkout <ветка pull request >
- Можно выгрузить правки локально из ветки Pull Request и протестировать отдельно. 
Но я рекомендую настроить тесты в CI/CD, это можно попросить сделать нашего бота. В том числе можно делать тесты для UI.
- Я про тестирование полной сборки до merge. Ее же где то собрать нужно перед тестированием. https://t.me/c/2975819706/1/55416
- Тестирование должно проходить в CI/CD, если это веб версия, то должны быть e2e тесты через playwright/puppeteer, я рекомендую использовать мою библиотеку: https://github.com/link-foundation/test-anywhere и https://github.com/link-foundation/browser-commander
- Рефакторинг нужно делать только в случае если у тебя код был на 100% покрыт тестами.
